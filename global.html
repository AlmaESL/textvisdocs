(async) barchart(tokensOrPath, containerId, nWordsDisplayed, sortBy, language, keywords, useTFIDF) → {void}

A plugin that renders a bar chart from an array of tokens or an array of text files.
- A list of kewywords can be searched for via the keyword input argument.
- Sort the bar chart by frequency or by alphabetical order.

TF-IDF scores are normalized if more than one txt file or token array were given in the input.

Prerequisites:
- A div container with an ID and given dimensions to place the bar chart in.
Parameters:
Name Type Default Description
tokensOrPath Array Array of token arrays, array of file paths to .txt files (strings) or single path to .txt file (string)
containerId String The name of the HTML container to hold the chart
nWordsDisplayed int The number of words to display. If not specified (or set to null), displays all
sortBy String frequency Sort output by 'frequency' or 'alphabetical', defaults to 'frequency'
language String en Language of the txt file, 'en' or 'sv', defaults to 'en'
keywords Array Array of words to search for, can be left empty []
useTFIDF bool false If true, sort bars based on TF-IDF score, defaults to false

The bar chart visualization
Type
void

// Call bar chart with an array of file paths, language set to english, number of words to display = 20, keyowrds sorted 
// alphabetically (useTFIDF set to false)
barchart([path1, path2], 'container', 20, 'en', 'alphabetical, ['hello', 'world', ...], false)

(async) normalizeTfIdfScores(processedDocuments) → {Array.<Object>}

A plugin that normalizes TF-IDF scores in the range of [0, 1] using min-max scaling. This plugin takes an array of processed documents, each containing term scores from calculateTfIdf , and scales these scores so that they fall within the range of [0, 1]. This normalization is based on the minimum and maximum scores found in the dataset, adjusting the scores linearly.
Parameters:

processedDocuments Array.<Object> An array of objects, each representing a document's term scores

(async) table(tokensOrPath, containerId, language, sortType, order, keywords, exp, shadowOn) → {Array}

A plugin that renders a table from an array of tokens or an array of text file paths.
- Sort the table by frequency, alphabetically or from TF-IDF scores, ascending or descending.
- Download the table from browser as excel file with the export argument set to true.

TF-IDF scores are normalized if more than one text file or token array have been provided.

Prerequisites:
- A div container with an ID to place the table in.
Parameters:
Name Type Default Description
tokensOrPath Array Array of token arrays or paths to txt files
containerId String The name of the container to hold the table
language String en Language of the text file, defaults to 'en'
sortType String frequency Sort type: 'alphabetically', 'frequency' or 'tfIdfScore', defaults to 'alphabetically'
order String descending Sort order: 'ascending' or 'descending', defaults to 'ascending'
keywords Array Array of words to search for and display in the table, can be left empty []
exp Boolean false Boolean variable that will export the table to an excel file if set to true, defaults to false
shadowOn Boolean false Boolean variable to turn shadow styles on or off, false=off by default

(async) treemap(tokensOrPath, containerId, Language, keywords, useTFIDF, shadowOn, cap, colorPaletteopt) → {Array}

A plugin that renders a treemap from an array of token arrays or an array of text file paths.
- A list of keywords can be searched for via the keywords input.
- TF-IDF scores can be computed by setting useTFIDF to true -> if more than one token array or text file,
normalized scores over entire corpus will be computed, else scores over the single document are computed.
Treemap also returns found keywords in an array, the array is empty if no keywords were found

Prerequisites:
- A div container with an ID, to place the treemap in.
Parameters:
Name Type Attributes Default Description
tokensOrPath Array Array of token arrays or paths to txt files
containerId String The name of the container to hold the treemap
Language String Language of the txt file, 'en' or 'sv', defaults to 'en'
keywords Array Array of words to search for, can be left empty []
useTFIDF Boolean Boolean for activating TF-IDF computations, defaults to false
shadowOn Boolean Boolean for activating shadows on the treemap, defaults to false
cap Number Integer than limits words displayed in treemap based on count, defaults to 1
colorPalette Array <optional>
 ['#A6CEE3', '#63A3CC','#1F78B4'] Array of hex colors for the treemap
Source:
  • textVisBundle.js, line 404
Returns:
An array of found keywords, will be 0 if no keywords were found
Type
Array
Example
// Call treemap with an array of token arrays, language set to english, an array with keys to search for and useTFIDF=true
treemap([tokens1, tokens2], 'container', 'en', ['hello', 'world'], true); 

// Call treemap with array of text file paths, language set to swedish, an array of keys to search for and useTFIDF=false
