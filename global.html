(async) barchart(tokensOrPath, containerId, nWordsDisplayedopt, sortByopt, languageopt, keywordsopt, useTFIDFopt) → {void}

A plugin that renders a bar chart from an array of tokens or an array of text files
- A list of kewywords can be searched for via the keyword input argument
- Sort the bar chart by frequency or alphabetical order

TF-IDF scores are normalized if more than one txt file or token array were given in the input.

Prerequisites:
- A div container
- Scripts wherein this plugin is called must be type="module"
Parameters:

containerId String The name of the HTML container to hold the chart

 The number of words to display. If not specified (or set to null), displays all.
sortBy String <optional>
 Sort output by 'frequency' or 'alphabetical'. Defaults to 'frequency'.
language String <optional>
 Language of the txt file, 'en' or 'sv' (if input is not tokenized)
keywords Array <optional>
 Array of words to search for
useTFIDF bool <optional>
 If true, sort bars based on TF-IDF score

(async) normalizeTfIdfScores(processedDocuments) → {Array.<Object>}

A plugin that ormalizes TF-IDF scores in the range of [0, 1] using min-max scaling. This function takes an array of processed documents, each containing term scores, and scales these scores so that they fall within the range of [0, 1]. This normalization is based on the minimum and maximum scores found in the dataset, adjusting the scores linearly.

Prerequisites:
- The input processedDocuments must come from the plugin calculateTfIdf
- Scripts wherein this plugin is called must be type="module"
Parameters:
Name Type Description
processedDocuments Array.<Object> An array of objects, each representing a document's term scores.
Source:

Array.<Object>

(async) sort(tokens, sortTypeopt, orderopt, wordFrequenciesopt) → {Map}

A function that sorts an array of tokens based on the specified criteria; 'alphabetically', 'frequency' or 'TF-IDF score'.
Note! TF-IDF based sorting computes and sorts by non-normalized TF-IDF scores -> sort for a corpus will sort after
TF-IDF scores within each separate document or token array

Prerequisites: Scripts wherein this plugin is called must be type="module"

order string <optional>
 "ascending" Sort order: "ascending" or "descending". Defaults to "ascending".
wordFrequencies Map <optional>
 Map of word frequencies. If not provided, sort function will compute frequencies.

(async) treemap(tokensOrPath, containerId, languageopt, keywords, useTFIDFopt, shadowOnopt, capopt, colorPaletteopt) → {Array}

A plugin that renders a treemap from an array of token arrays or an array of text file paths
- a list of keywords can be searched for via the keywords input
- TF-IDF scores can be computed setting useTFIDF to true -> if more than one token array or text file,
normalized scores over entire corpus will be computed, else scores over the single document are computed
Treemap also returns found keywords in an array, the array is empty if no keywords were found

Prerequisites:
- A div container with an ID, to place the treemap in
- Scripts wherein this plugin is called must be Scripts wherein this plugin is called must be type="module"
Parameters:
Name Type Attributes Default Description
tokensOrPath Array Array of token arrays or paths to txt files
containerId String The name of the container to hold the treemap
language String <optional>
 'en' Language of the txt file, 'en' or 'sv'
keywords Array Array of words to search for
useTFIDF Boolean <optional>
 false Boolean for activating TF-IDF computations
shadowOn Boolean <optional>
 false Boolean for activating shadows on the treemap
cap Number <optional>
 1 Integer than limits words displayed in treemap based on count
colorPalette Array <optional>
 ['#deebf7', '#9ecae1', '#3182bd'] Array of hex colors for the treemap
Source:
  • textVisBundle.js, line 415
